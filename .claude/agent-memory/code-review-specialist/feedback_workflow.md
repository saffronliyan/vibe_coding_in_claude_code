description: CLAUDE.md requires a todo.md plan, manual user approval, and small incremental steps before any code changes
type: feedback
---

Rule: Before making any code changes, write a checklist plan in task/todo.md. Ask the user to approve it manually. Execute one step at a time, give a high-level explanation per step, and add a review summary section at the end of todo.md.

**Why:** Explicitly mandated in CLAUDE.md. User wants to stay in control of the change cadence and review each step.
**How to apply:** Every implementation task in this repo must follow this workflow. Never skip straight to edits. Never batch large rewrites.
